# 🚀 คู่มือการใช้งาน MuMupow (Premium Setup Guide)
โปรแกรมควบคุมหน้าจอและรันบอทจำลองหน้าจอ Emulator พร้อมกันหลายจอเวอร์ชันพรีเมียม (ภาษาไทย 100%)

---

## 📢 วิธีการอัปเดตโปรแกรมเป็นเวอร์ชันใหม่ (สำหรับผู้ใช้เก่า) ⚠️ สําคัญมาก

หากคุณเคยใช้งาน MuMupow เวอร์ชั่นเก่าอยู่แล้ว และต้องการอัปเดตเป็นเวอร์ชันล่าสุดที่มีระบบสลับดึง OTP สำรอง และปุ่มแก้ไขบัญชี:

1. **อัปเดตตัวโปรแกรม (`MuMupow.exe`)**:
   - นำไฟล์ **`MuMupow.exe`** ตัวใหม่ ไปวางทับ (Replace) ตัวเดิมในโฟลเดอร์รันโปรแกรมของคุณได้ทันที
   - **ข้อมูลเดิมไม่หาย**: ข้อมูลรายการบัญชีเดิม (`accounts.json`) และพอร์ตเดิม (`ports.json`) จะยังอยู่ครบถ้วนและใช้งานต่อได้โดยอัตโนมัติ 
2. **การย้าย/เพิ่มมาโคร (`macros/`)**:
   - นำไฟล์มาโคร (ไฟล์นามสกุล `.json` เช่น `มาโคร_รีรหัสสมัครใหม่.json`) ไปใส่ไว้ในโฟลเดอร์ **`macros`**
   - คุณสามารถเลือกย้ายเฉพาะไฟล์มาโครที่ต้องการใช้งานได้เลย ไม่จำเป็นต้องเอาไปทั้งหมด
3. **การย้ายรูปภาพสแกนหน้าจอ (`templates/`)**:
   - หากมีการใช้มาโครตัวใหม่ที่มีคำสั่งตรวจจับรูปภาพ (เช่น ปิดโฆษณา) ให้นำรูปภาพกากบาทหรือหน้าจอหลักไปวางไว้ในโฟลเดอร์ **`templates`** ของเครื่องด้วย

## 🔐 ฟีเจอร์หลักเด่นในเวอร์ชันนี้: ระบบบัญชีรีรหัส & การดึง OTP สำรอง

เวอร์ชั่นนี้อัปเดตมารองรับระบบดึง OTP สำหรับบัญชีที่ซื้อมาใหม่ (Outlook/Hotmail) ผ่าน API ของ `read-mail.me` แบบไม่ติดล็อก และแก้ปัญหากรอกผิดได้ในตัว

### 1. วิธีนำเข้าบัญชีและบัญชีรีรหัส (Batch Import)
เมื่อซื้อบัญชีรีรหัสมาเป็นชุด คุณสามารถคัดลอกทั้งหมดมาวางนำเข้าได้โดยใช้ปุ่ม **"📥 นำเข้าบัญชีแบบกลุ่ม (Batch)"**:
* **รูปแบบการนำเข้าปกติ**: `อีเมล|รหัสผ่าน` หรือ `อีเมล|รหัสผ่าน|กลุ่ม`
* **รูปแบบสำหรับบัญชีรีรหัส (OTP API)**: `อีเมล|รหัสผ่าน|Refresh_Token|Client_ID`
  *(ระบบจะทำการแยกเก็บโทเคนและไคลเอนต์ไอดีเพื่อนำไปใช้ดึง OTP ผ่าน API ของ read-mail.me ทันทีเมื่อเรียกคำสั่ง)*

### 2. ปุ่มแก้ไขบัญชีใหม่ (`📝`)
หากคุณหรือลูกค้ากรอกข้อมูลผิด หรือต้องการอัปเดตโทเคน/รหัสผ่านใหม่:
* ในแท็บ **👥 จัดการบัญชี** จะมีปุ่มรูปดินสอ `📝` ปรากฏอยู่ข้างๆ ปุ่มลบ `X` ของแต่ละบัญชี
* คลิกที่ปุ่ม `📝` ข้อมูลของบัญชีนั้นจะถูกป้อนเข้าช่องกรอกด้านขวาโดยอัตโนมัติ
* ปุ่มหลักจะเปลี่ยนเป็นสีส้ม **"💾 บันทึกการแก้ไข"** (และมีปุ่มยกเลิกสีแดงขึ้นมา) ให้คุณแก้ไขแล้วกดเซฟได้เลยโดยไม่ต้องลบสร้างใหม่

### 3. ลำดับการทำงานของระบบดึง OTP อัตโนมัติ (Auto Fill OTP)
เมื่อบอททำงานถึงขั้นตอนกรอก OTP บอทจะตัดสินใจทำงานตามลำดับดังนี้:
1. **ดึงผ่าน Web API (สำรอง)**: หากบัญชีนั้นมีค่า `Refresh Token` และ `Client ID` บอทจะเชื่อมต่อ API ของ `read-mail.me` เพื่อดึงรหัสผ่าน OAuth2 ทันที (มีความเสถียรสูง ไม่ติดหน้าต่างบล็อกของ Microsoft)
2. **ดึงผ่าน IMAP (หลัก)**: หากไม่มีข้อมูลโทเคน บอทจะเชื่อมต่อเข้ากับเซิร์ฟเวอร์ `outlook.office365.com` ผ่าน SSL เพื่อเข้ากล่องเมลตามปกติ

### 🧩 4. ระบบชุดคำสั่งย่อยประกอบสคริปต์ (Script Sets) เหมือนต่อเลโก้
ฟีเจอร์ใหม่ที่ช่วยให้คุณแยกกลุ่มขั้นตอนการทำงานเป็นชุดสั้นๆ (เช่น เซ็ตล็อกอิน, เซ็ตปิดโฆษณา, เซ็ตเคลียร์กิจกรรม) แล้วนำมาเลือกต่อเรียงกันในมาโครหลักได้โดยไม่ต้องทำขั้นตอนยาวซ้ำๆ ในทุกโปรไฟล์:
* **วิธีบันทึกเป็น Set**:
  1. สร้างสคริปต์ขั้นตอนที่ต้องการในหน้าต่างหลักตามปกติ (เช่น ขั้นตอนการกรอกไอดี รหัสผ่าน และการเข้าสู่ระบบ)
  2. คลิกปุ่ม **"🧩 จัดการ Sets"** (อยู่ข้างๆ ปุ่มสร้างสคริปต์เร็ว)
  3. ตั้งชื่อที่กล่องขวามือ (เช่น `login_game`) แล้วกด **"➕ บันทึกขั้นตอนหลักปัจจุบันเป็น Set นี้"**
  4. ตัวชุดคำสั่งย่อยจะถูกเซฟแยกไฟล์ลงในโฟลเดอร์ `script_sets/`
* **วิธีนำ Set มาประกอบใช้งาน**:
  1. ในหน้าสร้างขั้นตอนมาโครหลัก เลือกประเภทคำสั่ง: **ใช้ชุดคำสั่ง (Run Set)**
  2. ระบุชื่อชุดคำสั่งที่คุณเซฟไว้ (เช่น `login_game`) ลงในช่อง **ข้อความ / แอป** แล้วกดเพิ่มขั้นตอน
  3. หรือคุณสามารถกดเปิดหน้าต่าง **"🧩 จัดการ Sets"** เลือกเซ็ตที่ต้องการ แล้วกดปุ่ม **"📥 แทรกเข้าคิวสคริปต์หลัก"** ได้ทันทีเช่นกัน
  * *ข้อดี: หากในอนาคตต้องการแก้ไขขั้นตอนของเซ็ตล็อกอิน คุณเข้ามาแก้ไขและเซฟที่ไฟล์ Set เพียงจุดเดียว โปรไฟล์มาโครทุกตัวที่เรียกใช้ล็อกอินนี้จะทำงานด้วยตรรกะใหม่ตามไปด้วยทันที!*
  * *ระบบตรวจจับการวนลูป (Anti-Cycle System): ตัวสแกนจะตรวจสอบความปลอดภัยในพื้นหลัง หากตรวจพบโครงสร้างที่อ้างอิงวนลูปกันเอง (เช่น Set A เรียก Set B และ Set B ดึง Set A กลับมา) ระบบจะแจ้งเตือนเพื่อป้องกันไม่ให้โปรแกรมค้าง*

## ⚙️ ขั้นตอนการเตรียมเครื่องก่อนเริ่มใช้งาน (สำคัญมาก)
เพื่อให้พิกัดและปุ่มกดต่างๆ ในเกมทำงานได้อย่างถูกต้อง แม่นยำ 100% กรุณาตั้งค่าโปรแกรมจำลองหน้าจอ Emulator (MuMu Player) ของคุณดังนี้:

### 1. ตั้งค่าขนาดหน้าจอจำลอง (Resolution)
1. เปิดตั้งค่าในโปรแกรมจำลองหน้าจอ (Emulator Settings)
2. ไปที่หัวข้อ **การแสดงผล (Display)**
3. ตั้งค่าแบบกำหนดเอง (Custom):
   * **ความกว้าง (Width):** `540`
   * **ความสูง (Height):** `960` (แนวตั้ง)
   * **ความหนาแน่นพิกเซล (DPI):** `160`

### 2. เปิดการตั้งค่าสำหรับนักพัฒนา (Developer Options)
1. ใน Emulator ไปที่ **ตั้งค่าระบบแอนดรอยด์** -> **เกี่ยวกับแท็บเล็ต/โทรศัพท์**
2. กดคลิกซ้ำๆ ที่ **หมายเลขบิลด์ (Build Number)** ประมาณ 7 ครั้งจนมีข้อความขึ้นว่า *"คุณเป็นนักพัฒนาซอฟต์แวร์แล้ว"*
3. ย้อนกลับไปที่เมนูหลักของตั้งค่า -> ไปที่ **ระบบ (System)** -> **ตัวเลือกสำหรับนักพัฒนาซอฟต์แวร์ (Developer Options)**
4. ติ๊ก **เปิดใช้งานการแก้จุดบกพร่อง USB (USB Debugging)**

## 📥 วิธีเปิดใช้งานโปรแกรมสำหรับผู้ใช้ทั่วไป
หลังจากที่ดาวน์โหลดโปรแกรมและทำการแตกไฟล์ ZIP เรียบร้อยแล้ว ให้ทำตามขั้นตอนดังนี้:
1. เปิดหน้าจอจำลอง Emulator (เช่น MuMu Player) ขึ้นมารอไว้กี่จอก็ได้ตามต้องการ
2. ดับเบิ้ลคลิกเปิดไฟล์ **`MuMupow.exe`**
3. โปรแกรมจะทำการสแกนหาหน้าจอจำลองทั้งหมดที่เปิดอยู่ในเครื่องอัตโนมัติ และจะแสดงชื่อเครื่อง/พอร์ตในแถบลิสต์ **"เลือก Emulator ที่จะควบคุม"** ด้านซ้ายมือ

## 📖 คู่มือการใช้งานฟีเจอร์หลักอื่นๆ

### 📂 1. ระบบจัดการกลุ่มบัญชี (Group Account Management)
ใช้สำหรับจัดสรรและจัดหมวดหมู่บัญชีบอทให้ค้นหาง่าย:
* กรอกข้อมูลบัญชีในช่อง และระบุชื่อ **กลุ่มบัญชี** (เช่น *Server 1*, *กลุ่ม A*, หรือ *ทั่วไป*)
* ข้อมูลจะถูกจัดเก็บลงในการ์ดด้านซ้ายโดยแยกเป็นโฟลเดอร์พับเก็บได้ตามกลุ่ม
* สามารถติ๊กถูกหน้ากลุ่ม เพื่อเลือก/ไม่เลือก บัญชีทั้งหมดในกลุ่มนั้นได้ในคลิกเดียว

### ⏸️ 2. โหมด "หยุดรอตรวจทานทีละชุด (Pause between sets)"
เหมาะสำหรับกรณีที่รันหลายจอพร้อมกัน และต้องการดูความถูกต้องก่อนข้ามไปทำเซ็ตถัดไป:
1. ติ๊กถูกที่ช่อง **"⏸️ หยุดรอตรวจทานทีละชุด (Pause between sets)"** ด้านขวาล่าง
2. บอทจะหยิบรหัสมาทำพร้อมกันเท่ากับจำนวน Emulator ที่เปิดอยู่ (เช่น เปิด 4 จอ ก็ทำทีละ 4 รหัส)
3. เมื่อจบรอบของชุดนั้น บอทจะหยุดรอและขึ้นปุ่มสีน้ำเงิน **"▶️ รันบัญชีชุดถัดไป (เหลือ X ไอดี)"** ให้ผู้ใช้ตรวจสอบจอ เมื่อพร้อมแล้วค่อยกดเพื่อรันชุดถัดไป

### 🖼️ 3. ระบบตรวจจับรูปภาพข้ามโฆษณาอัจฉริยะ (Image Match & Clear Ads Loop)
ใช้สแกนหารูปภาพบนหน้าจอด้วย OpenCV เพื่อปิดโฆษณาหรือป็อปอัปที่ขึ้นไม่สม่ำเสมอ:
* **เตรียมปุ่ม**: ครอปภาพปุ่มปิดหรือปุ่ม X ให้กระชับ แล้วนำไปเซฟไว้ในโฟลเดอร์ `templates`
* **ประเภทคำสั่งในมาโคร**:
  * **ตรวจจับรูปภาพ (Image Match)**: ค้นหารูปภาพที่ระบุ ถ้าพบระบบจะกดคลิกให้ทันที ถ้าไม่พบจะข้ามขั้นตอนไปทันทีโดยไม่เสียเวลารอ
  * **ลูปปิดโฆษณา (Clear Ads Loop)**: วนลูปปิดสิ่งกีดขวางไปเรื่อยๆ จนกว่าหน้าจอจะแสดงรูปภาพหลักเสร็จสิ้น (Lobby) (เช่น ระบุเป็น `lobby.png | close_btn.png,back.png`) บอทจะวนสแกนไม่เกิน 15 รอบ และหากไม่พบปุ่มใดๆ จะทำการจำลองส่งคำสั่งย้อนกลับ (BACK keyevent 4) ให้โดยอัตโนมัติ

### 🎯 4. วิธีค้นหาพิกัดปุ่มกดบนหน้าจอ (พิกัด X/Y)
หากไม่ทราบว่าปุ่มหรือตำแหน่งใดๆ บนจออยู่พิกัดใด:
1. ไปที่แท็บ **⚙️ ตั้งค่าโปรแกรม** -> กดปุ่ม **"🎯 เปิดแสดงเส้นพิกัดการจิ้ม"**
2. ใช้เมาส์คลิกบนหน้าจอ Emulator สังเกตตัวเลขแกน X และ Y ที่ขอบบนสุดของจอ Emulator (เช่น `X: 450 Y: 211`)
3. นำเลขพิกัดนั้นมากรอกใช้งานในขั้นตอนของมาโคร และเมื่อเสร็จแล้วสามารถเข้ามากดปิดแสดงผลได้

### 🖱️ 5. ควบคุมหน้าจอพร้อมกันแบบแมนนวล (Manual Command Sync)
ใช้ส่งคำสั่งฉุกเฉินหรือกดกิจกรรมแบบสดๆ พร้อมกันทุกจอ:
* ไปที่แท็บ **🖱️ สั่งการพร้อมกันแบบแมนนวล**
* พิมพ์รหัส ส่งคำสั่งคลิก เลื่อนจอ หรือส่งปุ่มระบบจำพวกปุ่ม Home, Back ไปยัง Emulator ทั้งหมดพร้อมกันแบบเรียลไทม์
